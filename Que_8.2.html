What is Semantic HTML and a list of commonly used semantic tags?

Semantic HTML refers to the use of HTML tags to convey the meaning or structure of the content on a web page, rather than just specifying its appearance. By using semantic tags, you provide additional context and information to both search engines and assistive technologies, making your web page more accessible, maintainable, and understandable. Here is a list of commonly used semantic HTML tags: - header: Represents the introductory content or a group of navigational links for a section or the whole page.
- nav: Represents a section containing navigation links.
- main: Represents the main content of a document or a specific section within it.
- article: Represents a self-contained composition that can be independently distributed or reusable, such as a blog post, news article, or comment.
- section: Represents a thematic grouping of content, typically with a heading.
- aside: Represents content that is tangentially related to the main content, such as sidebars, pull quotes, or advertisements.
- footer: Represents the footer of a document or a section, often containing information about the author, copyright, or contact details.
- figure: Represents self-contained content, such as images, diagrams, illustrations, or code snippets. = figcaption: Represents a caption or legend for a figure element.
- time: Represents a specific time or duration, typically used for dates, times, or time ranges.
- blockquote: Represents a section that is quoted from another source.
- cite: Represents the title or name of a work being cited.
- q: Represents a short inline quotation.
- code: Represents a fragment of computer code.
- pre: Represents preformatted text, such as code snippets or ASCII art.
